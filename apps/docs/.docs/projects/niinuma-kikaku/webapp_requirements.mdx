# 印刷会社向けWebアプリケーション要件定義書

## 1. 概要

### 1.1 目的
本Webアプリケーションは、ニイヌマ企画印刷の顧客向けサービスとして、印刷データの入稿から進捗確認、カスタマイズ注文までをオンラインで完結できる統合プラットフォームを提供することを目的としています。顧客の利便性向上と業務効率化を同時に実現し、競争優位性を確立します。

### 1.2 対象ユーザー
- 個人顧客（デザイナー、個人事業主等）
- 法人顧客（企業の広報・マーケティング担当者等）
- 代理店（デザイン会社、広告代理店等）

### 1.3 開発方針
- ユーザー中心設計（UCD）による直感的で使いやすいインターフェース
- モバイルファーストのレスポンシブデザイン
- シングルページアプリケーション（SPA）による高速なユーザー体験
- マイクロサービスアーキテクチャによる柔軟な拡張性
- 段階的なリリース計画によるフィードバックの早期取得と反映

## 2. 機能要件

### 2.1 共通機能

#### 2.1.1 認証・アカウント管理
- ユーザー登録機能
  - メールアドレス/パスワード登録
  - SNSアカウント連携（Google、Facebook等）
  - ビジネスアカウント登録（複数ユーザー管理）
- ログイン/ログアウト機能
- パスワードリセット機能
- 二要素認証（2FA）オプション
- アカウント情報管理
  - プロフィール情報編集
  - 請求先情報管理
  - 通知設定

#### 2.1.2 ダッシュボード
- 進行中の注文一覧（ステータス表示）
- 最近の注文履歴
- お知らせ・通知表示
- クイックアクション（新規注文、再注文等）
- カスタマイズ可能なウィジェット

#### 2.1.3 通知システム
- メール通知
- プッシュ通知（ブラウザ/モバイル）
- 通知センター（アプリ内）
- 通知設定カスタマイズ機能

### 2.2 顧客自動入稿システム

#### 2.2.1 プロジェクト管理
- プロジェクト作成・編集
- プロジェクトカテゴリ分類
- プロジェクトステータス管理
- プロジェクトアーカイブ

#### 2.2.2 入稿ウィザード
- 印刷物種類選択（名刺、チラシ、カタログ等）
- 仕様選択（サイズ、紙質、印刷方式）
- 部数・納期選択
- 特殊加工指定（箔押し、エンボス等）

#### 2.2.3 ファイル管理
- ドラッグ&ドロップファイルアップロード
- 大容量ファイル対応（〜1GB）
- 複数ファイル一括アップロード
- バージョン管理機能
- フォルダ構造によるファイル整理

#### 2.2.4 プリフライトチェック
- 解像度チェック（dpi確認）
- カラーモード確認（CMYK/RGB）
- トンボ・塗り足し確認
- フォント埋め込み確認
- エラー・警告表示と修正ガイド提供

#### 2.2.5 プレビュー機能
- PDF高精細プレビュー
- 3Dプレビュー（加工後イメージ）
- ページめくり機能（冊子の場合）
- ズーム・回転機能
- 印刷範囲確認機能

#### 2.2.6 修正・承認ワークフロー
- オンラインマークアップ機能
- コメント・フィードバック機能
- 修正依頼・再アップロード
- 承認フロー設定
- 複数審査者管理

### 2.3 リアルタイム進捗管理

#### 2.3.1 ステータストラッキング
- 入稿→校正→印刷→加工→発送の各工程表示
- リアルタイム状況更新
- 予定日時と実績の表示
- ボトルネック・遅延アラート

#### 2.3.2 タイムライン

- マイルストーン表示

#### 2.3.3 通知設定

- 重要マイルストーン到達通知
- 遅延警告通知
- 納品予告通知

#### 2.3.4 モバイル連携
- モバイルアプリ同期
- プッシュ通知
- QRコードによる案件紐付け
- オフライン確認機能

### 2.4 Canva連携機能

#### 2.5.1 製品カスタマイザー
- インタラクティブUIでのカスタマイズ

- 材質・サイズ・形状選択
- 特殊加工オプション選択

## 3. 非機能要件

### 3.1 UI/UX要件
- 直感的で使いやすいインターフェース設計
- 統一されたデザインシステム適用
- アクセシビリティ対応（WCAG 2.1 AA準拠）
- 多言語対応（日本語・英語）
- ダークモード対応（オプション）

### 3.2 パフォーマンス要件
- ページロード時間：2秒以内
- ファイルアップロード速度最適化
- 大規模ファイル処理の効率化
- 同時アクセス：最大500ユーザー
- レスポンスタイム：API応答0.5秒以内

### 3.3 セキュリティ要件
- SSL/TLS暗号化通信（HTTPS）
- データ暗号化（保存時・転送時）
- 認証・認可の厳格な管理
- WAF（Webアプリケーションファイアウォール）導入
- 脆弱性診断・ペネトレーションテスト実施
- GDPR/個人情報保護法対応

### 3.4 可用性要件
- サービス稼働率：99.9%以上
- バックアップ：日次フルバックアップ、1時間ごとの増分バックアップ
- 災害復旧計画（DRP）策定
- 最大許容ダウンタイム：計画メンテナンス以外で4時間以内

### 3.5 拡張性要件
- マイクロサービスアーキテクチャ採用
- コンテナ化技術（Docker, Kubernetes）活用
- 水平スケーリング対応
- APIファースト設計
- 機能拡張の容易性確保

### 3.6 互換性要件
- ブラウザ対応：Chrome, Firefox, Safari, Edge最新版
- デバイス対応：デスクトップ、タブレット、スマートフォン
- OS対応：Windows, macOS, iOS, Android
- 印刷データ形式：PDF, AI, PSD, INDD, EPS等主要形式
