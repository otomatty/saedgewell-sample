# Next.js Supabase Turboプロジェクトの実行

ローカルマシンでNext.js Supabase Turboプロジェクトを実行する方法を学びます。

プロジェクトを実行するには、以下の手順に従って開発サーバー、Supabase、およびStripe（課金システムのテスト用、オプション）を起動します。

## 1. 開発サーバーの起動

Webアプリケーションの開発サーバーを起動するには、以下を実行します：

```bash
# 開発サーバーの起動
pnpm dev
```

このコマンドはWebアプリケーションを起動します。

Webアプリケーションの詳細については、`apps/web/README.md`を参照してください。

### クイックスタート認証情報

MakerkitはデフォルトでいくつかのユーザーでDBをシードします。テストユーザーとしてログインするには、以下の認証情報を使用できます：

- メール：test@makerkit.dev
- パスワード：testingpassword

メールアドレスを確認するには、Inbucketにアクセスしてください。Supabaseは認証プロセス中に送信されたメールをキャプチャするためにInbucketを使用します。

頻繁に必要になるため、InbucketのURLをブックマークしておくことをお勧めします。

### スーパー管理者認証情報

スーパー管理者としてログインするには、以下の認証情報を使用できます：

```json
{
  "email": "super-admin@makerkit.dev",
  "password": "testingpassword"
}
```

スーパー管理者ユーザーにはMFAが必要なため、MFAを通過するには以下の手順に従ってください：

- **TOTP**：TOTPコードを生成するには、TOTPジェネレーターを使用してください。
- **シークレットキー**：TOTPコードを生成するには、テストシークレットキー`NHOHJVGPO3R3LKVPRMNIYLCDMBHUM2SE`を使用してください。
- **検証**：MFAコードを検証するには、TOTPコードとシークレットキーを使用してください。MFAコードを検証する際に、TOTPコードが期限切れでないことを確認してください。

## 2. Supabaseの起動

### Dockerを実行中にする

Supabase Dockerコンテナを起動するために、Dockerを実行中にする必要があります。Dockerが実行されていない場合は、今すぐ起動してください。

Dockerが実行中であることを確認し、以下のコマンドでSupabaseを起動します：

```bash
pnpm run supabase:web:start
```

このコマンドはローカルのSupabase Webサーバーを起動します。これにより、Supabaseにデプロイすることなくローカルで開発できます。

プロジェクトを本番環境にデプロイする準備ができたら、すべてが適切に設定されていることを確認するためにチェックリストに従ってください。

## 3. Stripeの起動（オプション）

課金システムをテストするには、以下のコマンドを実行してStripeを起動します：

```bash
pnpm run stripe:listen
```

このコマンドはWebhookをローカルマシンにルーティングします。

これで、ローカルマシンでNext.js Supabase Turboプロジェクトが実行されています。🚀

## 4. ローカル開発用のメールサーバー

ローカル開発中にメールを受信するために、SupabaseはInBucketを使用します。ローカル開発中は、SupabaseAuthと取引メールの両方からのメールをここで受信することになります。

頻繁に必要になるため、InBucketのURLをブックマークしておくことをお勧めします。

## 本番環境へのデプロイ

プロジェクトを本番環境にデプロイする準備ができたら、すべてが正しく設定されていることを確認するためにチェックリストに従ってください。

注意：Supabaseのホスティングインスタンスを使用することは、本番環境へのデプロイと似ています。したがって、すべてが適切に設定されていることを確認するために、チェックリストに従う必要があります。
