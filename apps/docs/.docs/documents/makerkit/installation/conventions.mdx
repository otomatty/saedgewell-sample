# Next.js Supabase Turboスターターキットの規約

Makerkitは、コードベースの一貫性と可読性を確保するために規約を使用しています。
以下は、Next.js Supabase Turboスターターキットで使用されている規約の一部です。

これらの規約に従う必要はありません：これらは単にコアキットで使用される標準的な実践セットです。もしこれらが気に入ったなら、キットを使用する際にこれらを維持することをお勧めします - あなたとあなたのチームメイトが理解できる一貫したコードスタイルを持つためです。

## Turborepoパッケージ

このプロジェクトでは、複数のアプリケーション間で共有できる再利用可能なコードを定義するためにTurborepoパッケージを使用しています。

- **Apps**は、ルーティング、レイアウト、グローバルスタイルを含む主要なアプリケーションを定義するために使用されます。
- **Apps**は設定をパッケージに渡し、パッケージは対応するロジックとコンポーネントを提供します。

## パッケージの作成

アプリコード用にパッケージを作成すべきですか？

**推奨事項**：複数のアプリケーションで再利用する予定がある場合や、ライブラリコードの作成に経験がある場合を除き、アプリコード用のパッケージを作成しないでください。
- アプリケーションが再利用を目的としていない場合は、すべてのコードを`app`フォルダに保持してください。このアプローチは時間を節約し、複雑さを軽減します。どちらも迅速な開発に有益です。
- **経験豊富な開発者**：経験がある場合は、必要に応じてパッケージを自由に作成してください。

## インポートとパス

パッケージやアプリからモジュールをインポートする場合は、以下の規約を使用してください：

- **パッケージから**：`@kit/package-name`を使用します（例：`@kit/ui`、`@kit/shared`など）。
- **アプリから**：`~/`を使用します（例：`~/lib/components`、`~/config`など）。

## 非ルートフォルダ

`app`ディレクトリ内の非ルートフォルダには、アンダースコアのプレフィックスが付いています（例：`_components`、`_lib`など）。

このプレフィックスは、これらのフォルダがルートではなく、共有コンポーネント、ユーティリティなどに使用されることを示しています。

## サーバーコード

`server`フォルダにあるファイルは、サーバーサイドコード専用です。クライアントサイドコードでは使用しないでください。

この規約は、コードが実行されることを意図している場所を明確にするのに役立ちます。これは、区別があいまいになる可能性があるNext.jsでは特に重要です。

例えば、アプリの一部に関連するサーバー関連のコードは、`_lib/server`フォルダにあります。

ファイルの先頭に`server-only`パッケージを含めて、誤ってクライアントサイドコードにインポートされないようにしてください。
