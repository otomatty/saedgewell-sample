# Next.js Supabase Turboスターターキットのコードベースのナビゲーション

Next.js Supabase Turboスターターキットのコードベースをナビゲートする方法を学びます。プロジェクト構造とコードベースの更新方法を理解します。

前述のように、このSaaSスターターキットはTurborepoのおかげでモジュール式になっており、単一のリポジトリで複数のパッケージを簡単に管理できます。

## プロジェクト構造

プロジェクトの主要なディレクトリは以下の通りです：

- **apps** - Next.jsアプリの場所
- **packages** - 共有コードとAPIの場所

### appsディレクトリ

ここにアプリ（または複数のアプリを追加する場合はそれらも）が配置されています。メインアプリはNext.jsアプリで、静的およびサーバーレンダリングされたWebアプリケーションを構築できるReactフレームワークです。

Next.jsアプリは以下を定義します：

- プロジェクトの設定
- アプリケーションのルート

### packagesディレクトリ

ここに共有コードとAPIが配置されています。共有コードは、Next.jsアプリとAPI両方で使用される関数のコレクションです。APIは、Supabaseデータベースとやり取りするために使用されるサーバーレス関数です。

共有コードは以下を定義します：

- 共有ライブラリ（Supabase、Mailers、CMS、Billing など）
- 共有機能（admin、accounts、account、auth など）
- UIコンポーネント（ボタン、フォーム、モーダルなど）

すべてのアプリは、packagesディレクトリからエクスポートされたAPIを使用および再利用できます。これにより、同じコードベース内に1つまたは複数のアプリを持ち、同じコードを共有することが容易になります。

## コードベースの詳細

コードベースは、ナビゲートと更新が容易になるように構造化されています。以下は、知っておくべき主要なファイルとディレクトリです：

```
- apps
  - web
    - app
    - components
    - config
    - lib
    - content
    - styles
    - supabase
```

### apps/webの詳細

apps/webディレクトリはNext.jsアプリが配置されている場所です。以下は、知っておくべき主要なディレクトリとファイルです：

- **app** - メインアプリが配置されている場所。ここでアプリケーションのルートを定義します。
- **components** - アプリケーションの共有コンポーネントを定義する場所。
- **config** - アプリケーションの設定を定義する場所。
- **lib** - アプリケーションの共有ライブラリを定義する場所。
- **content** - アプリケーションのコンテンツを定義する場所（デフォルトでは、Keystatic Markdocファイルを使用）。
- **styles** - アプリケーションのスタイルを定義する場所。
- **supabase** - Supabaseの設定、マイグレーション、テストを定義する場所。

### appの詳細

appディレクトリはアプリケーションのルーティングが定義されている場所です。以下は、知っておくべき主要なファイルです：

```
- app
  - home
  - (marketing)
  - auth
  - join
  - admin
  - update-password
  - server-sitemap.xml
```

ディレクトリを詳しく見ていきましょう：

- **home** - アプリケーションの内部ホームが配置されている場所。ユーザーがログインしているときのルートを定義します。
- **(marketing)** - アプリケーションのマーケティングページが配置されている場所。ユーザーがログインしていないときのルートを定義します。これはパスレスルートなので、URLには表示されません。
- **auth** - アプリケーションの認証ページが配置されている場所。ログイン、サインアップ、パスワード忘れページのルートを定義します。
- **join** - アプリケーションの参加ページが配置されている場所。このルートは、ユーザーが招待に従ってチームアカウントに参加できるルートです。
- **admin** - アプリケーションの管理ページが配置されている場所。スーパー管理者ページのルートを定義します。
- **update-password** - アプリケーションのパスワード更新ページが配置されている場所。これは、ユーザーがパスワードをリセットした後にリダイレクトされるルートです。
- **server-sitemap.xml** - アプリケーションのサイトマップが定義されている場所。

### homeの詳細

homeディレクトリはアプリケーションの内部ダッシュボードが配置されている場所です。以下は、知っておくべき主要なファイルです：

```
- home
  - (user)
  - [account]
```

ディレクトリを詳しく見ていきましょう：

- **(user)** - アプリケーションのユーザーページが配置されている場所。個人アカウントページのルートを定義します。
- **[account]** - アプリケーションのアカウントページが配置されている場所。チームアカウントページのルートを定義します。

homeパスにより、マーケティングページと内部ダッシュボードページを分離できます。

したがって、ユーザーのホームページは `/home/user` となり、アカウントのホームページは `/home/[account]` となります。
