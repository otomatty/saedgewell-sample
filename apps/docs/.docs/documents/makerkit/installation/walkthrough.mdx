# 機能ウォークスルー - Next.js Supabase Turboスターターキット

Next.js Supabase Turboスターターキットの機能ウォークスルーです。このガイドでは、キットの機能的な側面について学びます。

## Next.js Supabase Turboスターターキットの概要

Next.js Supabase Turboスターターキットは、以下の機能セクションに分けることができます：

- **マーケティング / 外部セクション** - アプリケーションの一般公開部分。ブログやドキュメントも含まれます。
- **認証** - アプリケーションの認証システム。
- **個人アカウント機能** - 個人アカウントで利用できる機能。
- **チームアカウント機能** - チームアカウントで利用できる機能。
- **招待** - アプリケーションの招待システム。
- **スーパー管理者** - アプリケーションのスーパー管理者機能。

## マーケティング / 外部セクション

マーケティングセクションは、アプリケーションの一般公開部分です。ユーザーが製品、価格設定、法的情報について学ぶことができる場所です。

ホームページはアプリケーションのランディングページです。製品の機能を紹介し、ユーザーにサインアップを促します。

### 価格設定

価格設定ページでは、ユーザーが製品のさまざまな価格プランについて知ることができます。

![価格設定ページ](./_images/pricing.jpg)

このセクションはホームページにも追加されています。

FAQページでは、ユーザーが製品に関する一般的な質問の回答を見つけることができます。

### お問い合わせ

お問い合わせページでは、ユーザーが会社に連絡することができます。ユーザーが直接会社にメッセージを送信できるお問い合わせフォームが含まれています。

ドキュメントは、ユーザーが製品の使用方法を学ぶことができる場所です。ガイド、チュートリアル、参考資料が含まれています。

## 認証

認証システムは、ユーザーがサインアップ、サインイン、パスワードのリセットを行える場所です。多要素認証も含まれています。

### サインアップ

サインアップページは、ユーザーがアカウントを作成できる場所です。メールアドレスとパスワードを提供する必要があります。

![サインアップページ](./_images/sign-up.jpg)

成功すると、ユーザーはメールアドレスの確認を求められます。これはデフォルトで有効になっており、セキュリティ上の理由から無効にすることはできません。

![メール確認ページ](./_images/sign-up-success.jpg)

### サインイン

サインインページは、ユーザーがアカウントにログインできる場所です。メールアドレスとパスワードを提供する必要があります。

![サインインページ](./_images/sign-in.jpg)

### パスワードリセット

パスワードリセットページは、ユーザーがパスワードをリセットできる場所です。メールアドレスを提供する必要があります。

![パスワードリセットページ](./_images/password-reset.jpg)

### 多要素認証

多要素認証（MFA）は、ユーザーがアカウントにサインインするために2つ以上の検証要素を提供する必要がある追加のセキュリティレイヤーです。

まず、ユーザーはMFAを有効にして要素を追加する必要があります：

![MFA設定ページ](./_images/setup-mfa.jpg)

その後、サインイン後、ユーザーは検証コードを提供する必要があります：

![MFA検証ページ](./_images/verify-mfa.jpg)

## 内部 / 認証後のページ

サインイン後、ユーザーはアプリケーションの内部ページにリダイレクトされます。これらのページは認証されたユーザーのみがアクセスできます。

アプリケーションの内部部分は2つのワークスペースに分かれています：

- ユーザーワークスペース
- チームワークスペース（オプション）

これはこのように機能します：

- **個人アカウント** - すべてのユーザーは個人アカウントを持っています。ここでは、設定の管理、チームアカウントの選択、オプションでプランへの登録、または提供する機能へのアクセスができます。
- **チームアカウント（オプション）** - ユーザーはチームアカウントを作成し、他のユーザーを招待して参加させることができます。チームアカウントには独自のワークスペースがあり、ユーザーはチーム設定、メンバー、請求を管理できます。

一般的に言えば、個人アカウントとチームアカウントでどの機能が利用可能かを決めるのはあなた次第です。個人アカウントの請求を無効にし、チームアカウントでのみ有効にする（またはその逆）を選択できます。

簡単な経験則は、個人アカウントは個人向け、チームアカウントは組織向けということです。個人アカウントを無効にすることはできません。それはユーザーがチームの一部でない場合にアプリケーションにアクセスできなくなるため、意味がありません。

## 個人アカウント機能

個人アカウントワークスペースは、ユーザーが個人アカウントで利用可能な機能にアクセスできる場所です。

これはログイン後のホームページであり、ユーザーワークスペースです：

- ホームページ - デフォルトでは空（ただし、オプションでユーザーが所属するチームのリストを提供できます）
- アカウント設定
- 請求（有効な場合）

### ユーザーワークスペースのホームページ

デフォルトでは、ユーザーホームは空のページです。これはおそらく、あなたのSaaSに固有のコンテンツを配置したいからです。

ただし、アカウントが所属するチームをリストするコンポーネントを提供しています。これはB2C SaaSよりもB2B SaaSに役立ちます。

内部ページには2つのレイアウトがあります：

- サイドバーレイアウト（デフォルト）
- トップナビゲーションレイアウト

どちらかを選択でき、ユーザーレイアウトまたはアカウントレイアウトのいずれかを選択することもできます。

以下はサイドバーレイアウトのユーザーホームページです：

![ユーザーホームページ（サイドバー）](./_images/user-home-sidebar.jpg)

そして以下はトップナビゲーションレイアウトのユーザーホームページです：

![ユーザーホームページ（トップナビゲーション）](./_images/user-home-header.jpg)

あなたのニーズに合ったものを選択できます。

### ユーザーワークスペースのアカウント設定

ナビゲーションから、ユーザーはアカウント設定にアクセスできます。ここでプロフィール情報の更新、パスワードの変更、言語、多要素認証などを行うことができます。

これまでライトモードを使用してきましたが、ダークモードはどうでしょうか？ダークモードに切り替えてみましょう：

![アカウント設定（ダークモード）](./_images/user-account-settings.jpg)

### ユーザーワークスペースの請求

ユーザーは請求ページからプランをチェックアウトして登録したり、請求ポータルにアクセスしたりできます。

これは請求が有効な場合にのみ表示されます。個人アカウントの請求を無効にし、チームアカウントでのみ有効にする（またはその逆）を選択できます。

![請求ページ](./_images/user-billing.jpg)

プランを選択すると、Stripe（またはLemon Squeezy）から埋め込みチェックアウトフォームが読み込まれます。

登録後、請求ページにはサブスクリプションの詳細が表示されます。

![サブスクリプション詳細](./_images/user-billing-plan.jpg)

## チームアカウント機能

プロフィールドロップダウンから、ユーザーは以下を選択できます：

- チームアカウントに切り替え
- チームアカウントを作成

![チームアカウント選択](./_images/user-profile-dropdown.jpg)

チームアカウントワークスペースでは、ユーザーは以下の機能にアクセスできます：

- チームアカウントホームページ：デフォルトでは、例としてモックダッシュボードを表示します。
- アカウント設定：ユーザーはチームアカウント設定を更新できます。
- メンバー：ユーザーはチームアカウントのメンバーを表示できます。
- 請求：ユーザーはチームアカウントの請求を表示できます。

### チームワークスペースのホームページ

デフォルトでは、チームホームは例としてのモックダッシュボードです。これを独自のダッシュボードや他のコンテンツに置き換えることができます。

![チームホームページ](./_images/team-account-dashboard.jpg)

### チームワークスペースのアカウント設定

ナビゲーションから、ユーザーはチームアカウント設定にアクセスできます。ここでチームアカウント情報の更新、チームアカウントの削除（所有者の場合）、またはチームアカウントからの退出（メンバーの場合）を行うことができます。

![チームアカウント設定](./_images/team-account-settings.jpg)

### チームワークスペースのメンバーページ

ナビゲーションから、ユーザーはチームアカウントのメンバーにアクセスできます。

ここでは以下のことができます：

- メンバーの表示
- 新しいメンバーの招待
- 既存のメンバーの削除または更新
- 所有権を別のメンバーに移譲
- 招待の削除または更新

![メンバーページ](./_images/team-account-members.jpg)

### チームワークスペースの請求

有効な場合、ユーザーはチームアカウントの請求を表示し、プランに登録したり請求ポータルにアクセスしたりできます。

![チーム請求ページ](./_images/team-account-billing.jpg)

### チームアカウントへの参加

ユーザーがチームアカウントに招待されると、招待リンク付きのメールを受け取ります。サインアップまたはサインイン後、参加ページにリダイレクトされます。

![招待メール](./_images/sign-up-invite.jpg)

#### 参加ページ

参加ページは、ユーザーがチームアカウントに参加できる場所です。

![参加ページ](./_images/join-team.jpg)

## スーパー管理者

スーパー管理者はSaaSの管理者です。SaaSのアカウントを管理できる特別な機能セットにアクセスできます。

### スーパー管理者のホームページ

ホームページはSaaSの小さな概要です。

このページをカスタマイズして、SaaSの最も重要なメトリクスを表示することができます。

![スーパー管理者ホームページ](./_images/super-admin-dashboard.jpg)

### SaaSのアカウント一覧

スーパー管理者はSaaSのすべてのアカウントを表示できます。個人アカウント、チームアカウント、またはすべてのアカウントでフィルタリングできます。

![アカウント一覧](./_images/super-admin-accounts.jpg)

### アカウント詳細の表示

スーパー管理者はアカウントの詳細を表示できます。アカウント情報、アカウントのメンバー、請求情報を確認できます。

さらに、以下のアクションを実行できます：

- アカウントの禁止（または禁止解除）
- アカウントの削除

![アカウント詳細](./_images/super-admin-account.jpg)
