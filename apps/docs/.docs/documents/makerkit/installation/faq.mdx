# FAQ - Next.js SaaSボイラープレートに関する質問

以下はこのキットに関する技術的なFAQです。Makerkitに関する一般的な質問については、Makerkit FAQを参照してください。

## 技術的なFAQ

### Next.js SaaSボイラープレートを使用するにはSupabaseを知っている必要がありますか？

はい、Next.js SaaSボイラープレートを使用するには、Supabaseの基本的な理解が必要です。以下のことを知っている必要があります：

- Supabaseプロジェクトの作成方法
- データベースのセットアップ方法
- PostgreSQLの理解
- Next.jsアプリでのSupabaseクライアントの使用方法

キットを使って学ぶことはできますが、Supabaseの使い方は教えていません。そのためには、[Supabaseのドキュメント](https://supabase.com/docs)を参照してください。

### Next.js SaaSボイラープレートを使用するにはNext.jsを知っている必要がありますか？

はい、Next.js SaaSボイラープレートを使用するには、Next.jsの基本的な理解が必要です。

### Supabaseを知りません！Next.js SaaSボイラープレートを購入すべきですか？

学習曲線に備えるか、先にSupabaseを学ぶことを検討すべきです。Next.js SaaSボイラープレートはSupabase上に構築されているため、Supabaseの使い方を知ることは不可欠です。

### Turborepoを知りません！Next.js SaaSボイラープレートを購入すべきですか？

はい、Turborepoの事前知識がなくてもNext.js SaaSボイラープレートを使用できます。Turborepoはボイラープレートのモノレポ構造を管理するために使用されています。あなたの主な焦点は、ボイラープレートを構築するツールではなく、apps/webディレクトリ内でSaaS製品を構築することになります。Turborepoを使用した経験がなくても、packagesディレクトリのコアコードをカスタマイズする予定がない限り、直接操作する必要はありません。

### 将来的に他のパッケージが追加されますか？

非常に可能性が高いです！このキットはモジュール式に設計されており、既存のコードを妨げることなく新機能やパッケージを追加できます。将来追加される可能性のある新しいパッケージや機能のアイデアは多数あります。

### このキットを非SaaSプロジェクトに使用できますか？

このキットは主にSaaSプロジェクト向けに設計されています。非SaaSプロジェクトを構築している場合、Next.js SaaSボイラープレートは過剰かもしれません。それでも使用することはできますが、SaaSプロジェクト特有の機能を削除する必要があるかもしれません。

### 個人アカウントのみを使用できますか？

はい、チームアカウントを無効にして個人アカウントのみを使用するための機能フラグを設定できます。

### このキットでReactパッケージXを使用できますか？

はい、このキットで任意のReactパッケージを使用できます。キットは単純なNext.jsアプリケーションなので、一般的には基盤となる技術（Next.js、Stripe、Supabaseなど）によってのみ制約され、キット自体によって制約されることはありません。すべてのコードを所有して編集できるため、キットをニーズに合わせて適応させることができます。ただし、基盤となる技術に制限がある場合は、それを回避する必要があるかもしれません。

### Makerkitは本番インスタンスをセットアップしてくれますか？

いいえ、Makerkitは本番インスタンスをセットアップしません。これには、Supabase、Stripe、その他必要なサービスのセットアップが含まれます。MakerkitはあなたのStripeやSupabaseアカウントにアクセスできないため、あなた側でのセットアップが必要です。Makerkitは、SaaSプロジェクトのセットアップを支援するコードベースとドキュメントを提供します。

### 問題が発生した場合、どのようにサポートを受けられますか？

サポートを受けるには、以下の方法があります：

- Discordにアクセスする
- サポートメールで連絡する

### サンプルプロジェクトやデモはありますか？

はい - OpenAIデモにアクセスできます。
