# ローカル開発へのアプローチ方法

前のセクションでは、リポジトリのクローン方法、環境変数の設定方法、アプリの外観のカスタマイズ方法、およびアプリ全体で使用する基本的なAPIについて学びました。

このセクションでは、アプリをローカルで開発する方法について学びます。

準備はいいですか？やることがたくさんあります！

一般的に、以下のことを行うことになります：

1. **カスタマイズ**: 環境変数（アプリケーション名、機能フラグなど）を設定します。これは、Makerkitをあなた自身のアプリに変えるための迅速な出発点です。

2. **データベース**: データベーススキーマの設計と記述。もちろん、あなたのアプリはデータを保存し、独自のスキーマを持つことになります。それを設計する時が来ました。

3. **ルーティング**: 新しいルートの追加。新しいページにはルートが必要です（アカウントの「ホーム」ページを再利用できる場合を除く）。

4. **データの取得**: DBからデータを取得し、新しいルートに表示します。

5. **データの書き込み**: 新しいフォームの追加。新しいデータを作成するためのフォームを追加する必要があります。

90%のケースでは、上記のことを行うことになります。残りの10%は、より複雑な新機能（非常に特殊な）を追加することであり、Makerkit自体とはあまり関係ありません。
