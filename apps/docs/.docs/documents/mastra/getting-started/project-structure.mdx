このページでは、Mastra でフォルダとファイルを整理するためのガイドを提供します。Mastra はモジュール式のフレームワークであり、モジュールを個別にまたは組み合わせて使用​​できます。

（クイックスタートで示したように）すべてを単一のファイルに記述することも、各エージェント、ツール、ワークフローをそれぞれのファイルに分割することもできます。

特定のフォルダ構造は強制していませんが、いくつかのベストプラクティスをお勧めします。CLI は適切な構造でプロジェクトを構築します。

**CLI の使用**

`mastra init` は、次のことができるインタラクティブな CLI です。

*   **Mastra ファイルのディレクトリを選択:** Mastra ファイルを配置する場所を指定します（デフォルトは `src/mastra`）。
*   **インストールするコンポーネントを選択:** プロジェクトに含めるコンポーネントを選択します。
    *   エージェント
    *   ツール
    *   ワークフロー
*   **デフォルトの LLM プロバイダーを選択:** OpenAI、Anthropic、Groq などのサポートされているプロバイダーから選択します。
*   **サンプルコードを含める:** 開始するのに役立つサンプルコードを含めるかどうかを決定します。

**プロジェクト構造の例**

すべてのコンポーネントを選択し、サンプルコードを含めることを前提とすると、プロジェクト構造は次のようになります。

**トップレベルフォルダ**

| フォルダ           | 説明                                   |
| :----------------- | :------------------------------------- |
| `src/mastra`       | コアアプリケーションフォルダ             |
| `src/mastra/agents`  | エージェントの構成と定義                 |
| `src/mastra/tools`   | カスタムツールの定義                     |
| `src/mastra/workflows` | ワークフローの定義                       |

**トップレベルファイル**

| ファイル           | 説明                               |
| :----------------- | :--------------------------------- |
| `src/mastra/index.ts` | Mastra のメイン構成ファイル             |
| `.env`             | 環境変数                             |
