# Linear のダウンロード

Linear は、ブラウザ、macOS、Windows、iOS、および Android で利用できます。

（Linear アイコンを示す画像）

## 概要

Linear は、デスクトップアプリケーション、ブラウザ、ネイティブモバイルアプリ、またはタブレット上のプログレッシブウェブアプリ (PWA) として利用できます。

最高の体験のために、デスクトップクライアントとモバイルアプリの使用をお勧めします。

### デスクトップクライアント

Linear は、macOS Intel、macOS Apple Silicon、および Windows で利用できます。 デスクトップアプリは linear.app/download からダウンロードしてください。

デスクトップアプリを使用する利点は次のとおりです。

*   通知はネイティブ OS と統合されています。 これは、Safari がブラウザ通知システム (PushAPI で作成) をサポートしていないため、Safari ユーザーにとって特に役立ちます。
*   Linear は常にオンになっており、ドックは未読の通知をすばやく通知します。
*   キーボードショートカットは、ブラウザベースのプリセットまたはカスタムショートカットと競合するリスクが少なくなります。

### ブラウザ

Linear ウェブアプリには、linear.app にログインしてアクセスできます。 Linear はブラウザウィンドウで直接起動します。

オフラインモードを含むデスクトップアプリのほぼすべての機能が Web で利用できます。 Linear はほとんどの最新ブラウザで動作するはずです。 Chrome または Safari をお勧めし、Firefox や Brave などの他の一般的なブラウザのサポートを維持するために最善を尽くしています。

*   **デスクトップアプリで開く:** このブラウザ設定では、Linear リンクがインストールされている場合はデスクトップアプリで開きます。ユニバーサルリンクは、ブラウザ版の [アカウント] > [設定] > [動作] > [デスクトップアプリで開く] から構成できます。

#### Brave Shields

Brave ユーザーは、linear.app ドメインの "Brave Shields" を無効にして、Web サイトがデスクトップアプリに自動的に接続して開けるようにする必要があります。

### iOS および Android

モバイルの Linear を使用すると、どこにいても作業を中断することができ、他のクライアントでの Linear の使用を補完できます。これらのアプリは App Store または Google Play Store からダウンロードするか、こちらで詳細をご覧ください。

両方のアプリで、タブは一般的なモバイルワークフローへのエントリポイントを提供します。

*   **ホーム:**
    *   [自分の課題] を通じて、割り当てられた課題、作成した課題、またはサブスクライブした課題を確認し、お気に入りにすばやくアクセスして、チームを探索して課題、サイクル、およびトリアージキューを表示します。
*   **受信トレイ:**
    *   受信トレイを確認し、アプリから直接コメントを送信したり、課題を更新したりして、通知に対するアクションを実行します。受信トレイは他のクライアントの Linear と同期し、通知の読み取り、スヌーズ、および削除をサポートします。
*   **課題を作成:**
    *   中央のタブから課題を作成します。カメラロールからメディアを添付してコンテキストを追加し、コードブロックや引用符などのリッチフォーマットを使用します。
*   **検索:**
    *   ワークスペース全体で課題またはプロジェクトを検索します。
*   **設定:**
    *   ワークスペースを切り替えたり、通知スケジュールをカスタマイズしたり、ライトテーマとダークテーマを切り替えたり、フィードバックを送信したりします。

PWA を使用して、ブラウザ経由でモバイルデバイスまたはタブレットデバイスで Linear を使用することもできます。 linear.app にアクセスしてサインインしてください。

## 問題がありますか?

### リアルタイム同期とオフライン

Linear は、すべての変更をリアルタイムで自動的に同期します。これは、チームがうまくコラボレーションし、迅速に作業する能力にとって非常に重要です。何らかの理由で変更をバックエンドに送信できない場合、アプリは変更をローカルに保存し、接続が復元されるたびに再試行します (例: インターネットが復旧した場合)。

アプリが同期を待機している変更が多いこと、または変更の送信に予想よりも時間がかかることを検出すると、サイドバーの上部にあるワークスペース名の横に「同期中」という単語が表示され始めます。その横の数字は、送信を待機している変更の数を示しています。接続を復元する前にアプリケーションを再起動した場合でも、それらはリロードされ、再試行されます。

オフラインモードは、完全な機能ではなく、フェイルセーフとして設計されています。

データの更新前に各変更の作成日は確認しません。つまり、オフラインモードで多くの編集を行うと、チームの誰かの変更を上書きする可能性があります (例: 課題の説明を編集したり、ステータスを更新したりした場合)。

### ローカルホストへのネットワークリクエスト

Linear デスクトップアプリがコンピューターにインストールされているかどうかを定期的に確認します。これを行うには、異なるポートでローカルホスト (コンピューター) にネットワークリクエストを送信します。各ポートチェックは、デスクトップアプリが実行されているかどうかを判断するのに役立ちます。これにより、特定の URL をデスクトップアプリで直接シームレスに開くことができ、ユーザーエクスペリエンスが向上し、Web アプリケーションとデスクトップアプリケーション間の統合が向上します。

確認するポートは 44450、18450、および 33234 です。

### アプリの自動更新

更新の場合、デスクトップアプリはバックグラウンドで自動的に更新をダウンロードしてインストールします。 MacOS の自動更新を無効にするには:

#### ターミナル

```bash
defaults write /Users/$USER/Library/Preferences/com.linear AutoUpdateDisabled -bool YES
```

自動更新を再度有効にするには、次のコマンドを使用します。

```bash
defaults write /Users/$USER/Library/Preferences/com.linear AutoUpdateDisabled -bool NO
```

#### Plist

MDM を介してデプロイされた `/Users/$USER/Library/Preferences/com.linear.plist` ファイルを使用すると、コンテンツは次のようになります。

```xml
<?xml version="1.0" encoding="UTF-8"?>
<!DOCTYPE plist PUBLIC "-//Apple//DTD PLIST 1.0//EN" "http://www.apple.com/DTDs/PropertyList-1.0.dtd">
<plist version="1.0">
<key>AutoUpdateDisabled</key>
<true/>
</plist>
```
