# 基本的なコンセプト

## ワークスペース(Workspace)

Linear ワークスペースは、個々の会社 (上の画像の Vast.craft) に関連するすべての課題、チーム、およびその他のコンセプトのコンテナです。

ユーザーとして、1 つまたは複数のワークスペースにアカウントを持つことができ、アプリケーションの左上隅にあるドロップダウンメニューを使用してこれらのアカウントを切り替えることができます。

ワークスペースにはそれぞれ `linear.app/example` のような形式の一意の URL があります。 Linear にログインするときは、特定のワークスペースにログインしています。

## チーム(Teams)

ワークスペース内には 1 つまたは複数のチームがあります。チームは通常、頻繁に連携して作業する人々のグループを表しますが、場合によっては、製品の一部としてチームを構成することを選択する場合があります。チームには課題が含まれており、チーム固有のプロジェクトを持つことができます（ただし、プロジェクトはチーム間で共有することもできます）。

チームは、サイクル、トリアージ、ワークフローなどの機能を構成するために利用できる独自の設定を備えているため、各チームの作業方法に最適になるように調整できます。各チームにはアーカイブも含まれており、バックログに積極的に保持されていない課題の記録が保持されます（アーカイブ規則はチーム設定で定義できます）。

## 課題(Issues)

Linear の最も基本的なコンセプトはIssue(課題)です。 

Linear の他のほとんどのコンセプトは、Issueに関連付けられているか、Issueをグループ化する機能があります。

Issueは、平易な言葉で説明されたタスクを表します。

各Issueは、単一の Linear チームに属している必要があり、チーム ID と番号で構成された識別子 ("ENG-123" など) があります。

Issueには、タイトルとステータスが必要です。

その他のすべてのプロパティと関係はオプションです。

これにより、Issueをすばやく作成し、不要な作業を削減できます（IssueはユーザーストーリーではなくIssueとして記述を参照）。

優先度、見積もり、ラベル、期日、担当者などの基本的なIssueプロパティをサポートしています。

Issue間の関係を作成して、Issueをブロックされている/ブロックしているとしてマークしたり、サブIssueを作成したりできます。

アプリのどこからでもIssueを作成できます。試してみてください。アプリの任意のページを開いて、`C` と入力します。

## ワークフロー(Workflow)

Issueの作業と完了が進むにつれて、チームごとに定義された順序付けられたIssueステータスのグループであるワークフローを介して移動します。

これらは、リストビューでIssueをグループ化し、ボードの列ヘッダーとして表示されるステータスです。

デフォルトのワークフローを作成し、設定でさらにカスタマイズできます。

Issueの現在の進捗状況を反映するために、Issueステータスを手動で更新できます。

ただし、可能な限り、統合または自動化を使用してIssueステータスを更新し、「チケットの管理」の必要性を回避することをお勧めします。

たとえば、GitHub および GitLab 統合では、git ブランチ名をコピーするとIssueが [進行中] に移動し、PR の作成、オープン、レビュー、およびマージの際にステータスの更新が継続されます。

サイクル自動化はIssueをバックログに出し入れし、リストが常に適切になるように、Issueを自動的にクローズしてアーカイブするように設定できます。

### 特殊なステータス

- **バックログ** は Linear のステータスカテゴリです。各チームには独自のバックログがあり、チームは課題をさらに細かくグループ化するために追加のバックログステータスを追加できます。
- **トリアージ** は Linear のオプションのステータスであり、課題を受け入れてチームのバックログまたはサイクルに移動する前に、課題をレビューできます。

# 課題の整理

## プロジェクト(Projects)

プロジェクトは、新しい機能の開始など、特定の期限付きの成果物に向けて課題をグループ化します。

プロジェクトには Linear に独自のページがあり、プロジェクトに関連するすべての課題、プロジェクトの詳細、および進捗状況とプロジェクトの完了日付範囲を示すグラフが表示されます。

プロジェクトは複数のチーム間で共有でき、イニシアチブまたはプロジェクトビューの下で整理できます。

## マイルストーン(Milestones)

マイルストーンは、個々のプロジェクト内の課題をさらに整理するために使用されるコンセプトです。

各プロジェクトについて、プロジェクトの完了における意味のある段階を表す方法でマイルストーンを定義し、プロジェクト内の課題をそれらのマイルストーンに割り当てます。

プロジェクトでマイルストーンでフィルタリングし、マイルストーンに属する課題に基づいて、特定の時点で個々のマイルストーンが完了にどれだけ近いかを簡単に確認できます。

## サイクル(Cycles)

サイクルはスプリントに似ており、特定の期間中に一連の課題を優先します。

サイクルは自動化され、繰り返されます。サイクルの開始日と期間を設定すると、N 週間ごとに繰り返されます。

これらは特にリリースで終了しません。サイクル中に完了しなかった課題は次のサイクルに繰り越され、見積もりのワークロードと完了率に関する基本的な統計が表示されます。

サイクルは必ずしもプロジェクトのようなテーマ別グループ化を持っている必要はなく、スケジュールに従って自動的に繰り返されますが、プロジェクトは意図的にキュレーションされます。

## 課題ビュー(Issue Views)

ビューは、一連のフィルターに従って課題をグループ化します。

チームのバックログ、すべて、またはアクティブな課題など、いくつかの課題ビューは標準で付属しており、Linear で利用可能な任意のフィルターに基づいてビューを定義できます。

ビューはフィルターに基づいているため、動的です。課題がビューを定義するフィルターセットを満たすと表示され、これらの基準を満たさなくなるとビューから離れます。

ビューは、自分自身、特定のチーム、またはワークスペース全体にのみ表示されるように構築できます。

ラベルの下にあるすべての課題、ユーザーに割り当てられた課題、または特定のサイクルまたはプロジェクトの一部である課題を表示するには、ビューにアクセスします。
*   **自分の課題** は、割り当てられた課題、作成した課題、およびサブスクライブしている課題を、最も関連性の高い課題から順に表示するキュレーションされたビューのセットです。

カスタムビューを構築するには、既存のビューのフィルターを使用し、それらのパラメーターからカスタムビューを作成します。サイドバーの [ビュー] に移動して、最初からカスタムビューを作成することもできます。

## プロジェクトの整理

### プロジェクトビュー

プロジェクトビューは、チームのサイドバーからアクセスできます。プロジェクトは、プロジェクトのステータスに基づいてこれらのカテゴリに表示されます。ワークスペースレベルでプロジェクトを表示して、組織内のすべてのプロジェクトの概要を取得することもできます。

課題ビューと同様に、プロジェクトビューは動的です。プロジェクトがビューのフィルター基準を満たすとそこに表示され、基準を満たさなくなるとビューから離れます。プロジェクトビューは、ワークスペース内のプロジェクトの実際の状態を表します。

### イニシアチブ

イニシアチブはプロジェクトを編成し、会社の目標とその進捗状況を紹介します。各イニシアチブには、手動でキュレーションされたプロジェクトのリストが含まれており、イニシアチブはワークスペースレベルの単一のイニシアチブビュー内で表示されます。これにより、複数のプロジェクトと長期的なタイムラインにわたる高レベルの計画が可能になります。

リーダーシップとマネージャーは、Linear ワークスペースの最上位のイニシアチブビューを使用して、進行中の目標、目的、およびその進捗状況の概要をすばやく把握できます。

## アクションの実行

Linear で課題、プロジェクト、またはその他のデータの一部を変更する操作を行うたびに、アクションを実行しています。

### 仕組み

Linear は、ボタン、キーボードショートカット、コンテキストメニュー、またはコマンドラインでアクションを検索するなど、複数の方法でアクションを実行できるように設計されています。これにより、アプリで何かを実行する方法を簡単に把握し、常に同じパターンに従っているため、筋肉の記憶を構築できます。

たとえば、課題にラベル「バグ」を適用するとします。次のいずれかの手順に従って実行できます。

*   **マウス** 課題を開き、[ラベルの適用] ボタンをクリックします
*   **マウス** ボードまたはリストビューから課題を右クリックし、コンテキストメニューを使用します
*   **キーボード** キーボードショートカット L を使用します
*   **キーボード** コマンドメニュー Cmd/Ctrl K を開き、ラベルを検索します。

課題に何か他のことを行う (見積もりを追加する、優先度を設定する、ステータスを更新する、担当者を設定する、プロジェクトに追加するなど) には、ほぼ同じ手順に従います。イニシアチブまたはプロジェクトビューからプロジェクトに対して何かを行う場合も、同様の手順を実行します。

### 一括アクション

Linear では、一度に複数の課題を簡単に更新できます。課題をチーム間、サイクルやプロジェクトとの間で移動したり、ラベルを適用したり、ステータスを一度に数十または数百の課題に変更したりできます。 Cmd/Ctrl A はビュー上のすべての課題を選択し、前述の方法のいずれかを使用してアクションを実行できます。

[元に戻す] アクション (Cmd/Ctrl Z) は、ほとんどのアクションを元に戻します。

### キーボードショートカット

キーボードは Linear を使用する最も速い方法です。通常はキーボードショートカットを使用しない場合でも、課題の作成など、一般的なアクションのショートカットを学習することをお勧めします。これらは学習するのに最も役立ちます。

*   C：任意のビューで課題を作成する
*   Cmd/Ctrl K：コマンドメニューを開く
*   x：選択する
*   Shift + 上/下 または Shift + クリック：複数を選択する
*   Esc：戻るまたはリストの選択をクリアする

現時点では、Linear でショートカットを再マップすることはできませんが、将来的にはカスタムショートカットの追加を検討する可能性があります。

キーボードショートカットパネルを示す GIF

### クイックナビゲーション

ショートカットはパターンに従うため、すばやく学習できます。ビュー間を移動するには、Gの後に *ショートカットを使用します。受信トレイに移動するには G の後に I 、現在のサイクルに移動するには G の後に V 、バックログに移動するには G の後に B を使用します。

### 簡単なアクセス

O の後に *ショートカットを使用すると、リスト上のアイテムを開くメニューが起動します。お気に入りを表示するには O の後に F 、プロジェクトを開くには O の後に P 、サイクルを開くには O の後に C を使用します。

ショートカットの完全なリストを表示するには、? を押します。

### コマンドメニュー

コマンドメニューを使用すると、ビューまたは選択に適用可能なアクションを検索して実行できます。

macOS で Cmd K 、Windows または Linux で Ctrl K を使用してコマンドメニューを起動します。

コマンドメニューのアクションは、状況に応じて異なります。たとえば、サイクルを表示していてコマンドメニューを開いた場合、コマンドメニューは最初にサイクルに関連するコマンドを表示します。

### マウス

もちろん、マウスで操作やアクションを実行することもできます。アプリ全体にボタン、メニュー、ドロップダウンがあります。リストやボードの課題、またはタイムライン上のプロジェクトを右クリックすると、コンテキストメニューが表示されます。
